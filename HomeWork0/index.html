• git config --global user.name blabla - встановлює ім'я, яке буде відображатися, як автор виконуваних коммітів
  • git config --global user.email blabal@bla - встановлює адресу електронної пошти, яка буде відображатися в інформації про ваші комміти
  • git init - створює новий локальний репозиторій з заданим ім'ям
  • git status - перераховує всі нові або змінені файли
  • git add "назва файлу або ." - індексує (підготовлює) вказаний файл для комміту
  • git commit (git commit -m "комент")- фіксує проіндексовані зміни і зберігає їх (коммітить зміни на локальному репозиторії)
  • git log - історія коммітов для поточної гілки
  • git checkout "назва старої версії або - " - переключитись на одну з старих версій файлів
  • git checkout -b "назва гілки" - створити гілку та переключитись на неї
  • git branch - виводить cписок гілок та вказує поточну гілку
  • git push - відправляє зміни в наш віддалений репозиторій
  • git pull - оновити локальний репозиторій з віддаленого
  • git merge "назва гілки" - вносить зміни зазначеної гілки в поточну гілку
  • git clone - скачує репозиторій разом з усією його історією змін
  • git add "file", git commit --amend -m "new comment" - зміна коменту для останнього комміту
  • git checkout "назва файлу" - скасувати зміни файлу, які не були проіндексовані (not git add)
  • git reset "назва файлу" - сказувати індексацію змін (after git add)
  • git revert - скасовуємо зміни після небажаного комміту
  • git rebase - перебазування (схоже на merge), використовується для короткочасних, локальних гілок
  • git cherry-pick "Коміт1 Коміт2 ..." - скопіювати серію комітів до поточного розташування
  • git stash
